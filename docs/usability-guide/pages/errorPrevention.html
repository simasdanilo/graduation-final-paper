Prevenção de erros

Erros ocorrem quando seu aplicativo não consegue completar uma ação ou quando o resultado de uma ação do usuário não corresponde ao resultado esperado. Assegure-se de que o seu aplicativo seja a prova de erros tanto quanto possível. Projete seu aplicativo para interrupções e para uma atenção fragmentada do usuário.

Minimize erros criando aplicativos que facilitam a entrada de informações de forma flexível. Os aplicativos devem aceitar formatos de dados comuns que usam recursos para evitar erros e melhorar a compreensão do usuário, como é o caso das máscaras de texto.

Uma máscara de texto é utilizada para o campo "telefone".

Inclua restrições quando for conveniente limitar as opções do usuário até que uma ação seja tomada ou o estado atual mude para um estado aceitável, como por exemplo você pode bloquear o botão de envio de um formulário até que o erro informado seja consertado.

Um número de telefone inválido faz o botão "REGISTRAR" permanecer desabilitado até que um número de telefone válido seja inserido.

Valorize as ações do usuário guardando os estados e salvando as informações inseridas por ele. Caso um erro ocorra faça com que seu aplicativo seja capaz de retornar ao último estado estável ou recuperar as informações salvas.

Uma mensagem que não foi enviada é salva como rascunho para ser enviada depois, caso o usuário assim deseje.

Projete seu aplicativo considerando cenários prováveis como: instabilidade na conexão, acabar a bateria do dispositivo, o dispositivo ser desligado ou o usuário realizar uma ação indesejada.

Confirmação

Confirmar envolve pedir ao usuário que verifique se ele deseja prosseguir com uma determinada ação. A confirmação ajuda a evitar que os usuários cometam erros dos quais possam se arrepender.

A confirmação não é necessária quando as consequências de uma ação são reversíveis ou desprezíveis. Por exemplo, se uma marca de seleção mostrar que uma imagem foi selecionada, a confirmação adicional é desnecessária.

Avise sobre consequências perigosas e confirme ações destrutivas. Use diálogos para confirmar ações do usuário difíceis de desfazer ou que são destrutivas, como as ações de "Excluir" ou "Remover".

Um diálogo pede uma confirmação para excluir os itens selecionados.
